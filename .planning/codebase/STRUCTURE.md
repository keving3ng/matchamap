## Directory Purposes

**`frontend/src/components/`:**
- Purpose: All React components (UI, pages, layout)
- Contains: 70+ .tsx files organized by feature
- Key files: `App.tsx` (root), `AppRoutes.tsx` (routing), `MapView.tsx`, `ListView.tsx`, `DetailView.tsx`
- Subdirectories: `ui/` (design system), `admin/` (admin panel), `auth/` (login), `passport/` (stamps)
- Tests: Co-located in `__tests__/` subdirectory

**`frontend/src/components/ui/`:**
- Purpose: Reusable UI components (buttons, badges, dialogs, skeletons)
- Contains: Button.tsx, Badge.tsx, AlertDialog.tsx, ErrorAlert.tsx, Skeleton.tsx, etc.
- Rules: No business logic; styling only; enforces 44px touch targets
- Used by: All other components; never use inline styles instead

**`frontend/src/components/admin/`:**
- Purpose: Admin panel pages and forms (editor-only)
- Contains: CafeManagementPage.tsx, EventManagementPage.tsx, StatsPage.tsx, etc.
- Auth: Protected by ProtectedRoute + requireAdminAuth backend middleware
- Lazy-loaded: Not bundled with main app; only loaded when admin route accessed

**`frontend/src/hooks/`:**
- Purpose: Custom React hooks for logic reuse
- Contains: useLazyData.ts, useGeolocation.ts, useLeafletMap.ts, useCafeSelection.ts, etc.
- Rules: One hook = one concern; no hook interdependencies; always use hooks (no higher-order functions)
- Tests: Mocked API, stores, browser APIs in `__tests__/`

**`frontend/src/stores/`:**
- Purpose: Zustand state management (global + persisted state)
- Contains: authStore.ts, dataStore.ts, locationStore.ts, cityStore.ts, visitedCafesStore.ts, etc.
- Rules: Focused per store; no store-to-store dependencies; use persist middleware for localStorage
- Tests: Mock API calls; test state transitions and persistence

**`frontend/src/utils/`:**
- Purpose: Pure utility functions (no React, no side effects)
- Contains: api.ts (centralized API client), distance.ts, hoursFormatter.ts, sanitize.ts, etc.
- Rules: No component imports; testable in isolation; memoize expensive calculations if needed
- Tests: Unit tests with sample data; no mocks needed

**`frontend/src/styles/`:**
- Purpose: Global CSS, Tailwind config, design tokens
- Contains: index.css (global styles), spacing.ts (design tokens)
- Design tokens: spacing.cardPadding (16px), spacing.minTouchTarget (44px), zIndex values
- Rules: Use tokens instead of arbitrary values; no component-scoped styles (use Tailwind classes)

**`frontend/src/constants/`:**
- Purpose: Non-code content (copy strings, field definitions)
- Key file: `copy.ts` — centralized user-facing strings (i18n-ready)
- Rules: ALL hardcoded strings → COPY constant; never inline text in components
- Example: `COPY.map.getDirections`, `COPY.admin.cafeForm.title`

**`frontend/src/config/`:**
- Purpose: Build-time configuration (feature flags)
- Key file: `features.yaml` — boolean flags with separate dev/prod values
- Usage: `useFeatureToggle('ENABLE_PASSPORT')` reads at runtime
- Rules: No secrets; regenerates on build

**`frontend/src/test/`:**
- Purpose: Test infrastructure and mocks
- Contains: setup.ts (globals, mocks), helpers.ts (test utilities), mocks/ (API/Leaflet mocks)
- setup.ts: Configures Vitest, mocks window.matchMedia, localStorage, etc.

**`frontend/src/types/`:**
- Purpose: Frontend-specific TypeScript types
- Contains: index.ts (component prop interfaces), json.d.ts (JSON module declaration)
- Rules: Import shared types from `../../shared/types`; define frontend-only types here

**`backend/src/routes/`:**
- Purpose: API endpoint handlers
- Contains: cafes.ts, drinks.ts, events.ts, auth.ts, admin-*.ts, etc.
- Pattern: Export async function handlers; accepts (request, env) → Promise<Response>
- Middleware: Routes accessed after middleware chain (auth, rate limit)

**`backend/src/middleware/`:**
- Purpose: Request/response processing
- Contains: auth.ts (JWT validation), rateLimit.ts, securityHeaders.ts, httpsOnly.ts
- Pattern: Middleware wraps handlers, can short-circuit with error Response
- Order applied in index.ts: HTTPS → CORS → rate limit → auth → route

**`backend/src/utils/`:**
- Purpose: Shared backend utilities
- Contains: response.ts (response formatters), validation.ts, sanitize.ts, auth.ts, cors.ts
- response.ts: jsonResponse(), badRequestResponse(), errorResponse() — consistent JSON format
- auth.ts: validateJWT(), generateToken(), verifyPassword()

**`backend/src/validators/`:**
- Purpose: Input validation schemas and helpers
- Contains: index.ts, comment.ts, review.ts, etc.
- Used in: Route handlers before processing user input

**`backend/drizzle/schema.ts`:**
- Purpose: SQLite table definitions (Drizzle ORM)
- Contains: cafes, drinks, events, users, userProfiles, waitlist, stats, etc.
- Indexes: createdIdx, deletedIdx, cafeIdx for query performance
- Migrations: Auto-generated in drizzle/migrations/ directory

**`shared/types/index.ts`:**
- Purpose: Type-safe API contract
- Contains: Cafe, Drink, Event, User, CityKey interfaces
- Usage: Import in frontend and backend for consistency
- Rules: Add types before implementing API endpoints

## Naming Conventions

**Files:**

| Type | Pattern | Example |
|------|---------|---------|
| Components | PascalCase | `CafeCard.tsx`, `MapView.tsx` |
| Hooks | camelCase, 'use' prefix | `useGeolocation.ts`, `useLazyData.ts` |
| Utilities | camelCase | `distance.ts`, `sanitize.ts`, `api.ts` |
| Stores | camelCase, 'Store' suffix | `authStore.ts`, `dataStore.ts` |
| Tests | Match source + `.test.ts` | `distance.test.ts`, `CafeCard.test.tsx` |

**Directories:**

| Pattern | Example |
|---------|---------|
| Feature-based | `components/admin/`, `components/auth/`, `components/passport/` |
| Type-based | `components/ui/`, `utils/`, `hooks/`, `stores/` |
| Test co-location | `src/__tests__/` at any level |

**Code (TypeScript):**

| Type | Pattern | Example |
|------|---------|---------|
| Components | PascalCase, exported | `export const CafeCard: React.FC<Props> = () => {}` |
| Functions | camelCase | `calculateDistance()`, `sanitizeInput()` |
| Constants | UPPER_SNAKE_CASE | `MAX_SEARCH_LENGTH`, `API_BASE_URL` |
| Interfaces | PascalCase, no 'I' prefix | `interface CafeCardProps`, `Cafe` |
| Types | PascalCase | `type UserRole = 'admin' \| 'user'` |
| Variables | camelCase | `isLoading`, `cafes`, `selectedCity` |

## Where to Add New Code

**New Feature (e.g., Reviews):**

1. **Frontend:**
   - Component: `frontend/src/components/ReviewForm.tsx`
   - Store: `frontend/src/stores/reviewStore.ts` (if shared state needed)
   - Hook: `frontend/src/hooks/useReviews.ts` (if complex logic)
   - Utility: `frontend/src/utils/reviewHelper.ts` (pure functions)
   - Tests: `frontend/src/components/__tests__/ReviewForm.test.tsx`
   - Copy: Add strings to `frontend/src/constants/copy.ts`

2. **Backend:**
   - Route: `backend/src/routes/reviews.ts` (handlers)
   - Validation: `backend/src/validators/review.ts` (input validation)
   - Middleware: Use existing auth, rate limit (in index.ts)
   - DB: Add table to `backend/drizzle/schema.ts`
   - Types: Add interface to `shared/types/index.ts`

3. **Tests:**
   - E2E: `tests/e2e/reviews.spec.ts` (Playwright)
   - Integration: `tests/integration/reviews.test.ts` (Vitest)

**New Component/Module:**

- **UI Component:** Place in `frontend/src/components/` with props interface
  - Should be small, reusable, and focused
  - Add to test file `frontend/src/components/__tests__/{name}.test.tsx`
  - Use Tailwind + design tokens; no inline styles

- **Page Component:** Place in `frontend/src/components/` with route registration in `AppRoutes.tsx`
  - Lazy-load via `React.lazy()`
  - Add `Suspense` boundary with loading skeleton

- **Shared Component:** If used in 2+ pages, place in `frontend/src/components/`
  - Extract to `frontend/src/components/ui/` if it's a design system element

**Utilities:**

- **Pure Functions:** `frontend/src/utils/{domain}.ts` (distance.ts, validation.ts, etc.)
- **Backend Utilities:** `backend/src/utils/{concern}.ts` (auth.ts, sanitize.ts, validation.ts)
- **Shared:** No shared utilities folder; put in `shared/types/` for types only

**Testing:**

- **Unit Tests:** Co-located `__tests__/` folder at component/hook/store level
  - File structure mirrors source: `SomethingComponent.test.tsx` next to `SomethingComponent.tsx`
  - Use mocking and helpers from `frontend/src/test/`

- **Integration Tests:** `tests/integration/` for cross-component or API-level testing
- **E2E Tests:** `tests/e2e/` for Playwright browser automation

## Special Directories

**`frontend/public/`:**
- Purpose: Static assets served as-is (index.html, favicon, manifest)
- Generated: No
- Committed: Yes
- Rules: Only index.html; all images in src/ for Vite optimization

**`frontend/dist/`:**
- Purpose: Built output (Vite build)
- Generated: Yes (build script)
- Committed: No (.gitignore)
- Contents: Optimized JS, CSS, images; ready for Cloudflare Pages

**`frontend/coverage/`:**
- Purpose: Test coverage reports
- Generated: Yes (npm test -- --coverage)
- Committed: No (.gitignore)
- View: Open `frontend/coverage/lcov-report/index.html` in browser

**`backend/.wrangler/`:**
- Purpose: Cloudflare Workers local dev state
- Generated: Yes (wrangler dev)
- Committed: No (.gitignore)
- Contains: Local D1 database, R2 bucket, cache state

**`backend/drizzle/migrations/`:**
- Purpose: SQL migration files (auto-generated)
- Generated: Yes (drizzle-kit generate)
- Committed: Yes (version control)
- Usage: Applied via `wrangler d1 migrations apply`

**`docs/`:**
- Purpose: Project documentation
- Generated: No
- Committed: Yes
- Key files: PRODUCT_FOUNDATION.md, SIMPLIFICATION_PLAN.md, TECH_SPEC.md, TESTING.md

**`.planning/codebase/`:**
- Purpose: Generated GSD codebase analysis documents
- Generated: Yes (via /gsd:map-codebase)
- Committed: Yes (for reference)
- Contents: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, STACK.md, etc.

**`.github/workflows/`:**
- Purpose: CI/CD automation (GitHub Actions)
- Generated: No
- Committed: Yes
- Files: build.yml, test.yml, deploy.yml, etc.

**`.husky/`:**
- Purpose: Git hooks (pre-commit, pre-push)
- Generated: No (configured in package.json)
- Committed: Yes
- Hooks: TypeScript check, linting on commit
